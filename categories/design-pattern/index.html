[design pattern] singleton pattern

2024-08-25

什麼是 Singleton 模式？Singleton 模式是一種設計模式，確保一個類在應用中只有一個實例，並提供全域訪問點。這樣可以防止重複創建相同的物件，節省系統資源，並確保共享資源或狀態的一致性。 Singleton 模式的應用場景 配置管理 確保配置對象在整個應用中是唯一的，允許在運行時修改配置。 日誌記錄 (Logging) 保證日誌系統的實例唯一性，統一管理應用的運行狀態。 資料庫連接池 (Database Connection Pool) 管理共享的資料庫連接池，避免創建多個不必要的連接。 狀態管理 (State Management) 集中管理應用程式的全局狀態，保持一致性。 Singleton 模式的代碼範例123456789101112131415161718192021..
